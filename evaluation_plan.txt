================================================================================
UniDebugger-Local: Evaluation Plan
================================================================================

All implementation changes are complete. This document describes what gets
recorded during evaluation and how to run a complete ablation study across
all 3 levels on QuixBugs (40 programs).

--------------------------------------------------------------------------------
WHAT GETS RECORDED PER RUN
--------------------------------------------------------------------------------

Each run creates two directories under res/quixbugs/, keyed by a deterministic
hash_id (8-char MD5 of {model_name, level}):

  Level 1  ->  hash_id: 8cd753df
  Level 2  ->  hash_id: 7b912ec5
  Level 3  ->  hash_id: c8484c57

RECORD FILES  (res/quixbugs/records/{hash_id}/)
------------------------------------------------

  params.json            Run parameters: {"model_name": "gpt-4o", "level": N}

  running.log            Full per-bug log with:
                           - SBFL hint: "Stack-trace SBFL: N suspicious lines"
                           - SpotBugs hint: "Suspected region: lines X-Y"
                           - Token counts per message
                           - HTTP call timestamps

  plausible_level1.txt   Bugs fixed at Level 1
                         (written in ALL level runs for comparison)

  plausible_level2.txt   Bugs fixed at Level 2
                         (written in Level 2 and Level 3 runs)

  plausible.txt          Bugs fixed at the HIGHEST level attempted
                         (= plausible_levelN.txt for the run's level)

  worked.txt             Bugs where any patch was generated (pass or fail)

  implausi.txt           Bugs tried but no passing patch found

  failed.txt             Bugs that errored out (no patch generated at all)

PRIMARY METRIC:
  Plausible fix rate = lines in plausible_level{N}.txt / 40

PER-BUG AGENT RESPONSES  (res/quixbugs/resp/{hash_id}/)
---------------------------------------------------------

  locator/
    aim/{bug}.java       Code annotated with "// buggy line" marks
    exp/{bug}.txt        Locator explanation
    ori/{bug}.txt        Raw LLM response

  fixer/
    aim/{bug}.patch      Generated unified diff patch
    exp/{bug}.txt        Fixer explanation
    ori/{bug}.txt        Raw LLM response

  slicer/                (Level 2+ only)
    aim/{bug}.java       Extracted suspicious 50-100 line segment

  summarizer/            (Level 2+ only)
    aim/{bug}.json       Code structure summary

  fixerpro/              (Level 2+ only)
    aim/{bug}.patch      FixerPro's refined patch

  helper/                (Level 3 only)
    aim/{bug}.txt        Local RAG debugging guide (replaces Tavily)

  repofocus/             (Level 3 only)
    aim/{bug}.json       Related files identified by RepoFocus


--------------------------------------------------------------------------------
RUN COMMANDS
--------------------------------------------------------------------------------

All commands from the src/ directory:

  cd /Users/Stanley3/Columbia_projects/Spring\ 2026/AI4Security/Unidebugger/src

Level 1 (Locator + Fixer only):
  python pipeline.py --model_name gpt-4o --data_name quixbugs --level 1 --limit 40

Level 2 (+ Slicer + Summarizer + FixerPro):
  python pipeline.py --model_name gpt-4o --data_name quixbugs --level 2 --limit 40

Level 3 (+ Helper RAG + RepoFocus) [full pipeline]:
  python pipeline.py --model_name gpt-4o --data_name quixbugs --level 3 --limit 40

Optional flags:
  --re_patch_num N    Number of patch attempts per bug (default: 2)
  --refinement        Enable refinement loop after failed patch


--------------------------------------------------------------------------------
COST ESTIMATE (gpt-4o pricing)
--------------------------------------------------------------------------------

  Level 1: ~2 API calls per bug x 40 = 80 calls  ~$0.40
  Level 2: ~4 API calls per bug x 40 = 160 calls ~$0.80
  Level 3: ~8 API calls per bug x 40 = 320 calls ~$1.60


--------------------------------------------------------------------------------
NOTES
--------------------------------------------------------------------------------

1. RESUMABLE: Runs are resumable. If interrupted, rerun the same command.
   Bugs already in worked.txt or failed.txt are automatically skipped.
   To restart from scratch, delete the hash directory:
     rm -rf res/quixbugs/records/8cd753df res/quixbugs/resp/8cd753df

2. EXISTING 5-BUG TEST: The 5-bug test run wrote to hash 8cd753df (Level 1).
   Those 5 bugs (BITCOUNT_1 through DETECT_CYCLE_1) will be SKIPPED if you
   run Level 1 again. Delete the directory for a clean full 40-bug run.

3. SECONDARY METRICS from running.log:
   - SBFL hit rate: grep "Stack-trace SBFL: [^0]" running.log | wc -l
   - SpotBugs hit rate: grep "SpotBugs found" running.log | wc -l
   - Average patch attempts: count fixer/aim/*.patch files per bug

4. VALIDATION: A "plausible" fix means all Gradle tests pass in the QuixBugs
   repo. Each patch is applied, tested, and reverted automatically.

--------------------------------------------------------------------------------
READING RESULTS QUICKLY
--------------------------------------------------------------------------------

After all 3 runs complete:

  for dir in 8cd753df 7b912ec5 c8484c57; do
    level=$(python3 -c "import json; d=json.load(open('res/quixbugs/records/$dir/params.json')); print(d['level'])")
    count=$(wc -l < res/quixbugs/records/$dir/plausible_level${level}.txt 2>/dev/null || echo 0)
    echo "Level $level: $count / 40 plausible"
  done

  Level 2: Y / 40 plausible
